# Audience Types

## end-user (default)

The person who uses the software to do their job, not to administer or develop it.

**Calibration:**
- Assumes zero technical knowledge.
- Explain what every button does, even if it seems obvious.
- Use everyday analogies for abstract concepts.
- Screenshot every screen.
- Avoid any mention of databases, APIs, servers, or code.
- Use "sign in" not "authenticate", "your account" not "your user record".

**Reading level target:** accessible to anyone who can use email.

---

## admin

A technical user who manages the software for others — configures settings, manages users, handles integrations.

**Calibration:**
- Assumes familiarity with web browsers and basic software concepts.
- Can reference configuration files, environment variables by name (briefly explain each).
- Include keyboard shortcuts.
- Note permissions / role requirements for each action.
- Screenshots optional for clearly-labeled UI elements; always include for complex or multi-step flows.

**Reading level target:** IT support, office manager, tech-savvy team lead.

---

## power-user

An expert user of the software who wants to work faster and use advanced features.

**Calibration:**
- Skip basics; open with advanced features.
- Lead with keyboard shortcuts table.
- Include tips for batch operations, automation, hidden settings.
- Omit or compress introductory chapters ("Getting Started" to one short paragraph).
- Assume the user already knows how to sign in and navigate the main UI.

**Reading level target:** someone who has used the software daily for 3+ months.
